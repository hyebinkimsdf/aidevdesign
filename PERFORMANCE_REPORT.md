## app/page.tsx 변경 내역

### 렌더링 방식

```
[변경 전]                          [변경 후]
─────────────────────────          ─────────────────────────
사용자 요청                         빌드 시 HTML 사전 생성
      ↓                                    ↓
서버에서 HTML 생성 (매번)           CDN에 캐시된 HTML 즉시 서빙
      ↓                                    ↓
사용자에게 전달                     사용자에게 전달
                                   TTFB 60–80% 단축
```

```tsx
// 변경 전 — 없음 (기본 SSR)

// 변경 후
export const dynamic = "force-static";   // 빌드 시 정적 HTML 생성
```

### Projects 컴포넌트 로딩

```
[변경 전]                          [변경 후]
─────────────────────────          ─────────────────────────
페이지 로드 시                      페이지 로드 시
모든 JS 한 번에 다운로드            Nav, Hero 등 필수 컴포넌트만 로드
                                          ↓
                                   Projects는 별도 청크
                                   (스크롤 시 필요할 때 로드)
초기 JS 번들 크기 ↑                 초기 JS 번들 크기 ↓
```

```tsx
// 변경 전
import Projects from "./components/Projects";

// 변경 후
const Projects = dynamicImport(() => import("./components/Projects"), {
  loading: () => <스켈레톤 UI />,   // 로딩 중 placeholder 표시
});
```

## 개선 항목

### 1. 페이지 로딩 방식 변경 (SSR → SSG)

**원인**  
Next.js 기본 구조는 요청이 올 때마다 서버에서 HTML을 새로 만드는 SSR 방식이다.  
포트폴리오처럼 내용이 바뀌지 않는 사이트에서는 불필요한 서버 처리 시간만 추가된다.

| | 변경 전 | 변경 후 |
|--|--------|--------|
| HTML 생성 시점 | 사용자가 접속할 때마다 | 배포 시 1회 미리 생성 |
| 응답 방식 | 서버 직접 응답 | CDN 캐시 서빙 |
| TTFB 예상 | 서버 상태에 따라 가변 | 60–80% 단축 예상 |

---

### 2. LCP 개선 — Hero 제목 애니메이션

**원인**  
`<h1>` 태그에 `opacity: 0`에서 시작하는 애니메이션이 걸려 있었다.  
브라우저는 opacity가 0인 동안 해당 요소를 LCP 후보로 인식하지 않아 LCP 측정이 지연됐다.

계산: FCP 1.9s + 딜레이 120ms + 애니메이션 650ms = **LCP 기준점 2.7s 이후**

| | 변경 전 | 변경 후 |
|--|--------|--------|
| h1 초기 상태 | opacity: 0 (애니메이션 대기) | opacity: 1 (즉시 표시) |
| 시각 효과 위치 | h1 태그 자체 | 내부 span으로 이동 |
| LCP 예상 개선 | 3.5s | 최대 1.0s 단축 → 2.5s 목표 |

---

### 3. LCP/FCP 개선 — 프로젝트 이미지 최적화

**원인**  
프로젝트 썸네일에 `unoptimized` 옵션이 설정돼 있어 Next.js의 이미지 최적화(WebP 변환, 크기 조정, 캐시)가 전혀 동작하지 않았다.

| | 변경 전 | 변경 후 |
|--|--------|--------|
| 이미지 포맷 | JPG 원본 그대로 전달 | WebP 자동 변환 |
| 크기 최적화 | 없음 | 뷰포트 기준 자동 조정 |
| 이미지 전송량 | 원본 크기 | 40–70% 감소 예상 |

---

### 4. INP 개선 — 검색/필터 응답성

**원인**  
검색어를 입력할 때마다 즉시 필터링 연산이 실행돼 타이핑이 버벅이는 현상이 있었다.

| | 변경 전 | 변경 후 |
|--|--------|--------|
| 필터 실행 시점 | 키 입력마다 즉시 | React가 여유 시간에 처리 |
| 적용 방식 | 동기 처리 | `startTransition` + `useDeferredValue` |
| 체감 효과 | 타이핑 시 화면 끊김 | 입력 즉각 반응, 결과는 뒤따라옴 |

---

### 5. INP 개선 — 스크롤 불필요한 리렌더링

**원인**  
스크롤할 때마다 Nav 컴포넌트가 React 상태를 업데이트해 불필요한 리렌더링이 반복됐다.

| | 변경 전 | 변경 후 |
|--|--------|--------|
| 상태 업데이트 빈도 | 스크롤 이벤트마다 | 스크롤 기준 결과가 바뀔 때만 |
| 리렌더링 횟수 | 매우 빈번 | 필요한 경우만 |

---

### 6. 접근성 — 색상 대비 (WCAG AA 기준)

**원인**  
어두운 배경(`#0c0c0c`)에 회색 텍스트(`#6b7280`)를 사용했고, 투명도까지 낮춰 기준치 미달이었다.

| 클래스 | 변경 전 대비율 | 변경 후 대비율 | 기준 |
|--------|--------------|--------------|------|
| `text-muted` | ~4.8:1 | ~5.8:1 | 4.5:1 ✅ |
| `text-muted/80` | ~3.8:1 | ~4.6:1 | 4.5:1 ✅ |
| `text-muted/75` | ~3.6:1 | ~4.4:1 | 4.5:1 ⚠️ |
| `text-muted/50` | ~2.4:1 | ~2.9:1 | 4.5:1 ❌ (장식용 처리) |

변경 내용: CSS 변수 `--muted` 값을 `#6b7280` → `#9ca3af`로 상향

---

### 7. GTM 스크립트 로딩 지연

**원인**  
Google Tag Manager 스크립트가 페이지 초기 로딩 시 실행돼 첫 상호작용 응답 속도에 영향을 줬다.

| | 변경 전 | 변경 후 |
|--|--------|--------|
| 로딩 시점 | 페이지 로드 직후 (`afterInteractive`) | 페이지 완전 로드 후 (`lazyOnload`) |
| 초기 INP 영향 | 스크립트가 메인 스레드 선점 가능 | 초기 상호작용과 경합 없음 |
