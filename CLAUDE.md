# CLAUDE.md

`@AGENTS.md`

이 저장소에서 Claude Code는 아래 원칙을 우선 적용한다.

## 목표

- 토큰을 아끼면서 빠르게 작업한다.
- 보안에 민감한 정보는 절대 노출하거나 기록하지 않는다.
- 큰 변경 전에는 범위를 줄여 확인하고, 작은 단위로 수정한다.

## 작업 방식

- 먼저 요청과 직접 관련된 파일만 읽는다.
- 전체 프로젝트를 한 번에 스캔하지 않는다.
- 긴 파일은 처음부터 끝까지 모두 출력하지 말고, 필요한 구간만 본다.
- 답변은 요약 위주로 짧게 작성하고, 코드 전체 복붙 대신 변경 핵심만 설명한다.
- 같은 내용을 다시 읽거나 다시 설명하지 않는다.
- 가능하면 기존 패턴을 재사용하고 새 추상화나 새 의존성 추가는 최소화한다.

## 파일 탐색 규칙

- 파일 찾기는 `rg --files` 와 `rg` 를 우선 사용한다.
- 읽기 전에는 관련 파일 후보를 먼저 좁힌다.
- 한 번에 많은 파일을 열지 않는다. 보통 1~3개씩 확인한다.
- 자동 생성 산출물, 빌드 결과, 락파일 전체 내용은 꼭 필요할 때만 본다.
- `node_modules` 는 Next.js 공식 가이드를 확인해야 할 때만 필요한 범위로 읽는다.

## 구현 규칙

- 요청 범위를 벗어나는 리팩터링은 하지 않는다.
- 작은 패치로 끝낼 수 있으면 큰 구조 변경을 하지 않는다.
- 이미 있는 스크립트(`npm run dev`, `npm run build`, `npm run lint`)를 우선 사용한다.
- 테스트나 검증도 요청과 관련된 최소 범위로만 실행한다.
- 문제를 설명할 때는 원인, 영향, 수정 내용을 짧게 정리한다.

## 보안 규칙

- `.env*`, API 키, 토큰, 쿠키, 개인 정보는 읽거나 출력하지 않는다.
- 비밀값이 필요한 작업은 값 자체를 요구하지 말고, 사용자가 직접 설정했다고 가정하고 진행한다.
- 로그, 예시, 문서에 실제 키나 민감한 값을 남기지 않는다.
- 권한 확대, 네트워크 사용, 패키지 설치, 외부 명령 실행은 꼭 필요할 때만 제안한다.
- 사용자가 요청하지 않은 민감 파일 수정은 하지 않는다.

## 확인이 필요한 상황

- 여러 구현 방식이 있고 되돌리기 비용이 큰 경우
- 새 패키지 설치가 필요한 경우
- 데이터 구조 변경, API 계약 변경, 관리자 기능 변경이 포함되는 경우
- 보안 또는 배포 영향이 있는 경우

## 이 프로젝트 기준

- Next.js 16 사용 중이므로 프레임워크 동작을 추측하지 말고 `AGENTS.md` 지침을 따른다.
- UI 수정 시 기존 컴포넌트 구조를 우선 존중한다.
- 관리자/문의 데이터 관련 작업은 실제 데이터 손상 없이 보수적으로 수정한다.

## 응답 스타일

- 불필요한 서론 없이 바로 결론부터 말한다.
- 목록은 짧게 유지한다.
- 사용자가 바로 다음 행동을 할 수 있게 필요한 정보만 준다.
